# 2. 核心概念：节点图的工作方式

在开始搭建更复杂的流程之前，我们先来理解 NodePy 的核心——**节点图**。它就像一张数据加工的“电路图”，每个节点完成一项特定任务，连线决定数据的流向。掌握了这些基础概念，你就能像搭积木一样灵活地构建各种数据处理流程。

通过本章，你将学会：
- **节点图的四大要素**：节点、端口、连线、数据流分别是什么，它们如何协同工作
- **类型系统**：NodePy 的八大数据类型，以及类型不匹配时如何快速定位问题
- **节点操作**：如何添加、配置、连接、运行节点，并查看运行结果

准备好了吗？让我们从节点图的构成开始，一步步揭开它的面纱。

---

## 2.1 什么是节点图（节点/端口/连线/数据流）

一个节点图由四个基本元素组成：

###  节点（Node）
节点是流程中的功能单元，每个节点负责一项具体操作，比如“读取表格”、“计算两列之和”、“绘制图表”。节点通常有一个**图标**、一个**名称**，以及若干个**端口**。

###  端口（Port）
端口是节点与外界交换数据的接口，分为**输入端口**（在节点左侧）和**输出端口**（在节点右侧）。  
- **输入端口**：节点运行时需要从这里获取数据。  
- **输出端口**：节点运行后产生的结果从这里流出。  
每个端口都有明确的**数据类型**（如 `Table`、`int`、`float` 等），只有相同类型的端口才能相连，否则会报错。

###  连线（Connection）
连线连接两个节点的端口，表示数据从输出端口流向输入端口。连线必须**同类型**，并且不能形成循环（目前不支持循环，但可以通过循环节点实现类似效果）。

###  数据流（Data Flow）
数据按照节点之间的连线方向流动：前一个节点的输出成为后一个节点的输入。当流程运行时，系统会自动从输入节点开始，沿着连线依次执行每个节点，直到所有节点处理完毕。

>  你可以把节点想象成工厂里的加工机器，端口是原料入口和产品出口，连线是传送带。原料从源头机器（输入节点）开始，经过一系列加工，最终产出成品（输出节点）。

## 2.2 类型系统与类型校验

NodePy 采用**严格的静态类型系统**，每个端口、每个参数都有明确的类型。这样做的好处是：**大部分错误能在运行前发现，避免流程跑到一半崩溃**。

### 八大基本类型
系统中共有八种数据类型，你会在端口和参数中频繁见到它们：

| 类型名     | 说明                                                         | 示例                  |
| ---------- | ------------------------------------------------------------ | --------------------- |
| `int`      | 整数，底层为 Python 的 `int` 或 `int64`                      | `42`, `-5`            |
| `float`    | 浮点数，底层为 Python 的 `float` 或 `float64`                | `3.14`, `-0.5`        |
| `bool`     | 布尔值，`True` 或 `False`                                    | `True`, `False`       |
| `str`      | 字符串                                                       | `"Hello"`, `"2024"`   |
| `Table`    | 表格，底层为 Pandas `DataFrame`，可包含多列、多行           | 见下方说明            |
| `File`     | 文件对象，存储在用户专属空间中（默认配额 5GB）               | 上传的 CSV、图片等    |
| `Datetime` | 日期时间，精确到秒                                           | `2024-01-01 00:00:00` |
| `Model`    | 机器学习模型，底层为 scikit-learn 估算器                     | 训练好的模型对象      |


### 表格中的列类型
表格（`Table`）本身是一种类型，但它内部的每一列也有自己的数据类型，允许以下五种列类型：
- `int`
- `float`
- `bool`
- `str`
- `Datetime`

### 类型校验规则
- **端口连接**：两个端口的数据类型必须**完全一致**，不允许任何隐式转换。例如，一个输出 `int` 的端口不能直接连接到需要 `float` 的输入端口。
- **节点参数**：在配置节点时，参数中允许有限的隐式转换：
  - `int` → `float`
  - 符合 ISO 8601 格式的 `str` → `Datetime`
- **表格列运算**：很多操作要求列类型匹配，比如两列相加要求两列都是数值类型（`int` 或 `float`）。

### 类型不匹配提示
当你尝试连接类型不匹配的端口，或者配置节点时选择了错误类型的列，系统会立即给出**红色错误提示**。

这种即时反馈能帮你快速修正错误，保证流程正确运行。

## 2.3 节点操作：添加、配置、连接、运行、查看输出

下面我们快速过一遍在画布上操作节点的基本步骤（很多你已经在上一个项目中实践过了）。

### 节点分类
节点按照功能分为多个分类，方便你快速定位所需功能：

- **输入**：提供数据来源，如表格、常量、文件、K线数据等。
- **计算**：执行数值、布尔运算，以及类型转换和列间计算。
- **控制**：实现循环、条件判断、自定义脚本等逻辑控制。
- **文件**：处理文件的上传、下载、格式转换。
- **字符串处理**：对字符串进行切片、替换、正则匹配、分词、情感分析等。
- **表格处理**：对表格进行筛选、排序、分组、连接、缺失值处理等操作。
- **日期处理**：日期时间的计算、转换、格式化。
- **分析**：统计分析、差分、滑动窗口、重采样、累积计算等。
- **可视化**：快速绘图、双轴绘图、统计绘图、词云、K线图。
- **机器学习**：特征工程、模型训练、预测、评估、聚类。

###  添加节点
1. 右键打开**节点菜单**，展开分类。
2. 找到需要的节点，用鼠标将其**拖拽**到中间的画布区域。
3. 释放鼠标，节点即出现在画布上。

###  配置节点
1. 节点上面可以看到它可以配置的参数。
2. 根据节点功能填写或选择参数。

###  连接节点
1. 将鼠标悬停在某个节点的**输出端口**上。
2. 按住鼠标左键，向外拖出一条线，直到目标节点的**输入端口**上。
3. 松开鼠标，连线即建立。如果连接合法（类型一致），连线保持正常颜色；否则会变红并提示错误。

>  你也可以从输入端口向输出端口拖拽连线，效果相同。要删除连线，只需选中它并按 `Delete` 键。

###  运行流程
NodePy 默认采用**自动运行**模式：每当节点图发生变化（如添加节点、修改配置、重新连线）时，系统会自动重新执行所有受影响的节点，确保数据是最新的。  
如果你想手动触发运行，可以点击画布右下角的 **“同步”** 按钮，所有节点将按顺序执行一次。

###  查看输出
要查看某个节点的运行结果：
**双击节点**，在弹出的侧边面板中，会显示该节点的**输出数据**。如果是表格，会以表格形式展示；如果是文件（如图表），会直接预览或提供下载链接。

##  小结

核心概念是节点式编程的基石：
- **节点、端口、连线、数据流**构成了流程的基本骨架。
- **严格的类型系统**保障了流程的可靠性，运行前就能发现类型错误。
- **添加、配置、连接、运行、查看输出**是日常操作五步曲。
