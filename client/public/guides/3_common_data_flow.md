# 3. 常见数据流程

掌握了节点图的基本操作后，我们来系统学习如何用节点搭建实际的数据处理流程。数据处理通常遵循一个固定的生命周期：从数据采集开始，经过清洗转换，再到分析与统计，最后进行可视化与输出。本章将沿着这条主线，带你认识 NodePy 中最常用、最实用的节点。

通过本章，你将学会：
- **数据采集**：如何从文件、手动输入、模拟生成或 K 线接口获取数据
- **数据清洗**：如何处理缺失值、重复值、过滤行、转换列类型
- **数据分析**：如何计算统计指标、处理时间序列、生成新指标列
- **可视化与输出**：如何快速绘图、制作专业图表，并导出结果文件
- **综合实战**：从 K 线数据到图表 + 导出的完整流程

每个部分都会介绍最常用的节点，并给出典型组合示例，让你能快速应用到自己的项目中。准备好了吗？让我们从数据采集开始。

---

## 3.1 数据采集

任何流程的第一步都是把数据“喂”进来。NodePy 支持多种数据来源：手动创建、文件上传、生成模拟数据等。

###  从文件读取表格
最常见的场景是读取本地的 CSV、Excel 或 JSON 文件。  
1. 拖入 **文件上传** 节点（位于 **文件** 分类）。  
2. 点击“上传文件”选择本地文件。节点输出一个 `File` 对象。  
3. 拖入 **文件转表格** 节点（**文件**），将 **文件上传** 的输出（`file`）连接到它的输入（`file`）。  
4.  **文件转表格** 节点无需额外配置（系统自动识别文件格式），输出即为 `Table` 类型。  

>  如果文件包含日期列，系统会自动尝试解析为 `Datetime` 类型，你也可以后续用 **日期解析** 节点（**日期处理**）手动转换。

###  手动创建表格
对于小规模数据或测试，可以用 **表格** 节点（**输入**）直接输入。  
- 配置 **rows**（行数据）、**col_names**（列名）和 **col_types**（列类型），就像我们在快速上手中做的那样。

###  生成模拟数据
- **随机表格**（**输入**）：生成指定行数的随机数列表。需要连接 **行数**（`int`）和可选的 **最小值/最大值**。  
- **范围表格**（**输入**）：生成等差数列或日期范围。需要连接 **start**、**end** 和可选的 **step**。  
- **常量** / **布尔** / **字符串** / **时间输入**（均在 **输入** 分类）：输出单个固定值，可作为其他节点的参数。

###  读取文本文件
如果需要处理纯文本（如日志、文章），使用 **提取文本** 节点（**文件**）。  
- 连接 **文件上传** 的输出到 **提取文本** 的输入，输出为 `str` 类型，可供后续字符串处理节点使用。

###  金融 K 线数据
**K线数据** 节点（**输入**）可以直接获取股票或加密货币的行情数据。  
- 配置 **data_type**（`stock` 或 `crypto`）、**symbol**（如 `"TSLA"`）、**interval**（时间间隔）。  
- 可选择在参数中固定起止时间，或通过输入端口动态传入。输出为包含标准 OHLCV 列的表格。

## 3.2 数据清洗与转换

原始数据往往“脏乱差”，清洗转换是流程中最耗时但也最关键的环节。以下节点帮你快速处理常见问题。

###  处理缺失值与重复值
- **删除缺失值**（**表格处理**）：按指定列删除包含空值的行。配置 **subset_cols** 选择要检查的列。  
- **填充缺失值**（**表格处理**）：用常数、前向填充或后向填充填补空值。支持多列分别指定填充值。  
- **去重**（**表格处理**）：根据一列或多列删除重复行，保留第一次出现的行。

###  过滤行
**过滤行** 节点（**表格处理**）根据某布尔列的值将表格拆分为两部分：`true_table`（条件为 True 的行）和 `false_table`（条件为 False 的行）。  
- 通常先通过 **列与常量比较**（**计算**）或 **列间比较**（**计算**）生成一个布尔列，再连接到此节点。

###  列操作
- **筛选列**（**表格处理**）：只保留需要的列，同时输出保留的表格和被丢弃的列组成的表格（可用于后续分析）。  
- **重命名列**（**表格处理**）：通过键值对（原列名 → 新列名）修改列名。  
- **数据类型转换**：**列转整数**、**列转浮点数**、**列转布尔值**、**列转字符串**（均在 **计算** 分类）。注意转换规则与普通类型转换一致。  
- **移动行**（**表格处理**）：将某列的数据向上或向下移动指定步数，常用于计算环比、同比（需配合其他节点）。

###  单列运算
- **数值运算**：**列一元运算**（**计算**）、**列二元运算**（**计算**）。  
- **字符串处理**：**批量首尾字符去除**、**批量字符串拼接**、**批量正则表达式匹配**（均在 **字符串处理** 分类）。  
- **正则提取**：**正则表达式提取** 节点（**字符串处理**）对单条字符串应用正则，返回匹配结果的表格（每行一个匹配，每列一个捕获组）。  
- **情感分析**（**字符串处理**）：对文本列进行情感打分（需先通过逐行处理，将在第4章介绍）。

## 3.3 数据分析与统计

清洗干净后，就可以对数据进行分析，提取有意义的指标。

###  统计汇总
**统计信息** 节点（**分析**）对单列计算多种统计量：计数、均值、标准差、最小值、最大值、总和、四分位数。  
- 输出为多个单独的数值端口，可连接到后续节点（如显示、比较）。  
- 如果需要对多列分别统计，可多次使用此节点。

###  时间序列处理
- **差分**（**分析**）：计算相邻行的差值，用于消除趋势或平稳性处理。  
- **滑动窗口**（**分析**）：计算移动平均值、移动总和等。需指定窗口大小、最小周期数和方法（mean、std、sum、min、max）。  
- **日期重采样**（**分析**）：将时间序列按指定频率（天、小时、分钟、秒）聚合，支持均值、求和等。  
- **累积计算**（**分析**）：累积和、累积积、累积最大值、累积最小值。

###  计算新指标列
- **数据变化率**（**分析**）：计算当前行相对于前一行的百分比变化（如增长率）。  
- **列间运算**（**计算**）：两列加减乘除生成新列，例如“利润率 = 利润 / 销售额”。  
- **移动平均**：使用 **滑动窗口**（**分析**）的 `mean` 方法。  
- **累积值**：使用 **累积计算**（**分析**）生成累积销售额等。

## 3.4 可视化与输出

分析完成后，需要将结果呈现出来或导出供其他系统使用。

###  快速图表
- **快速绘图**（**可视化**）：可在一个图表中绘制多条线、柱、散点、面积，每条系列可单独指定类型和 Y 轴（左/右）。非常适合探索性分析。  
- **双轴绘图**（**可视化**）：专门为左右双轴场景设计，左侧和右侧各支持一种图形类型（线图或柱状图）。

###  专业图表
- **统计绘图**（**可视化**）：基于 seaborn，支持多种统计图形：柱状图、计数图、散点图、箱线图、小提琴图、直方图等。可通过 **hue** 参数进行分组着色。  
- **词云**（**可视化**）：需要两列：词语列和频率列。生成词云图片。  
- **K线图**（**可视化**）：专门用于展示金融 K 线，需指定时间列、开盘/高/低/收/成交量列，支持中国红绿和美国配色。

###  导出结果
- **表格转文件**（**文件**）：将表格保存为 CSV、Excel 或 JSON 文件。可自定义文件名（无需扩展名，系统自动添加）。输出为 `File` 对象。  
- **文件上传** 节点（**文件**）也可用于下载（通过其输出的文件对象，可在前端下载）。

>  所有图表节点输出的都是 `File` 类型的图片（PNG），你可以直接双击预览并下载。

## 3.5 综合实战：从 K 线数据到图表 + 导出

下面我们用一个完整的例子，将本章学到的节点串联起来，实现一个真实的数据分析任务：**获取股票日线行情 → 计算移动平均 → 绘制图表 → 导出结果**。全程无需本地文件，所有数据通过内置节点获取。

### 步骤详解

1. **获取 K 线数据**  
   - 从左侧 **输入** 分类拖入 **K线数据** 节点。  
   - 配置节点参数：  
     - **data_type**：选择 `stock`（股票）或 `crypto`（加密货币），这里选 `stock`。  
     - **symbol**：输入股票代码，例如 `"TSLA"`（特斯拉）。  
     - **interval**：选择 `"1d"`（日线）。  
     - **start_time** 和 **end_time**：在参数中直接填入，例如 `"2024-01-01"` 到 `"2024-03-31"`（注意格式为 `YYYY-MM-DD`）。  
   - 节点输出一个表格，包含时间、开盘、高、低、收盘、成交量等列。

2. **筛选需要的列**  
   - 从 **表格处理** 分类拖入 **筛选列** 节点。  
   - 将 **K线数据** 节点的输出端口（`kline_data`）连接到 **筛选列** 的输入端口（`table`）。  
   - 配置 **筛选列** 节点参数，在 **selected_cols** 参数中勾选 `Open time`（时间）和 `Close`（收盘价）两列。  
   - 该节点输出两个表格：`selected_table`（只含时间与收盘价）和 `dropped_table`（被丢弃的其他列）。我们只需将 `selected_table` 用于后续步骤。

3. **计算移动平均**  
   - 从 **分析** 分类拖入 **滑动窗口** 节点。  
   - 将上一步的 `selected_table` 连接到 **滑动窗口** 的 `table` 输入。  
   - 配置节点参数：  
     - **col**：选择 `Close`（要计算的列）。  
     - **window_size**：输入 `5`（5 日移动平均）。  
     - **method**：选择 `mean`（平均值）。  
     - **result_col**：可自定义列名，例如输入 `MA5`。  
     - 其他参数保持默认。  
   - 节点输出表格，包含原始的两列（`Open time`, `Close`）以及新增的 `MA5` 列。

4. **绘制对比图表**  
   - 从 **可视化** 分类拖入 **快速绘图** 节点。  
   - 将上一步输出的表格（含 `Open time`, `Close`, `MA5`）连接到 **快速绘图** 的 `input` 端口。  
   - 配置节点参数：  
     - **x_col**：选择 `Open time`（作为 X 轴）。  
     - **y_col**：创建两个Y轴，分别选择 `Close` 和 `MA5`。  
     - **plot_type**：为两个 Y 轴系列都选择 `line`（折线图）。  
     - 其他参数（如图表标题）可留空或自定义。  
   - 运行后双击节点，即可预览收盘价与 5 日均线的对比折线图。

5. **导出结果表格**  
   - 从 **文件** 分类拖入 **表格转文件** 节点。  
   - 将步骤 3 输出的表格（或步骤 2 的筛选后表格，根据需求）连接到 **表格转文件** 的 `table` 输入。  
   - 配置节点：  
     - **filename**：可选，例如输入 `"TSLA_with_MA5"`（无需扩展名）。  
     - **format**：选择 `xlsx`（Excel 格式）或 `csv`。  
   - 节点输出一个 `File` 对象，双击该节点即可在预览页面下载文件到本地。

6. **运行与查看**  
   - 点击画布右下角的 **同步** 按钮（或等待自动运行）。  
   - 依次双击 **快速绘图** 和 **表格转文件** 节点，分别查看生成的图表和下载 Excel 文件。

>  你可以替换股票代码、时间范围、移动平均窗口，甚至增加更多技术指标（如将成交量加入图表），轻松复用此流程分析任意股票或加密货币。

通过组合这些节点，你几乎可以完成任何常规的数据处理任务。下一章我们将深入**逻辑控制与自动化**，学习如何实现循环、条件判断和自定义脚本，让流程更智能、更强大。
