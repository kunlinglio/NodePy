# 4. 逻辑控制与自动化

基础的数据处理流程是线性的：一个节点接一个节点，数据依次流过。但在实际项目中，我们常常需要更复杂的逻辑——比如对表格的每一行单独处理、在滚动窗口上计算、或者根据条件动态更新单元格。本章将介绍 NodePy 的**控制节点**，它们能让你的流程具备“智能”，实现循环、判断和自定义逻辑。

通过本章，你将学会：
- **逐行处理**：如何用 `ForEachRow` 对表格的每一行独立操作
- **滚动窗口处理**：如何用 `ForRollingWindow` 在时间序列上计算滑动窗口统计
- **列级映射**：如何用 `MapColumn` 高效地对单列进行逐单元格变换
- **单元格操作与行列转换**：如何用 `GetCell`、`SetCell`、`Unpack`、`Pack` 在循环体内精细操作数据
- **自定义脚本**：如何用 `CustomScriptNode` 编写 Python 代码实现任意逻辑
- **综合实战**：从 K 线数据到移动平均的完整滚动窗口流程

准备好了吗？让我们从逐行处理开始，一步步掌握这些强大的控制节点。

---

## 4.1 逐行处理（ForEachRow）

**用途**：对表格的每一行依次执行相同的操作，例如对每条文本进行情感分析、对每个数值进行复杂计算等。

### 节点组成
- **表格逐行循环**（**控制**）实际上是一对节点：**ForEachRowBegin** 和 **ForEachRowEnd**。当你从菜单拖入“表格逐行循环”时，这两个节点会同时出现在画布上，并自动用一条灰色的“循环体”虚线框连接在一起。

### 工作方式
- **Begin 节点**接收一个表格（`Table`），然后逐行输出（每次输出一行，仍为 `Table` 类型，但只有一行数据）到循环体内。
- 在 Begin 和 End 之间，你可以插入任意多个处理节点，对单行数据进行操作。
- **End 节点**收集所有处理后的单行结果，将它们重新组合成一个完整的表格输出。

---

## 4.2 滚动窗口处理（ForRollingWindow）

**用途**：在时间序列或有序数据上，对每个固定大小的窗口（如过去 7 天）进行计算，常用于移动平均、滚动统计等。

### 节点组成
- **滑动窗口循环**（**控制**）也是一对节点：**ForRollingWindowBegin** 和 **ForRollingWindowEnd**。拖入后自动成对出现。

### 工作方式
- Begin 节点接收一个表格，根据设定的 **窗口大小**（`window_size`）依次输出每个窗口（包含多行）到循环体内。
- 在 Begin 和 End 之间，你可以对每个窗口表格进行任意处理（例如计算窗口内的均值、最大值等）。
- End 节点收集每个窗口的处理结果（每个结果应为单行表格），最终合并输出。

---

## 4.3 列级映射（MapColumn）

**用途**：对表格中某一列的每个单元格进行独立变换，相当于 Excel 中“对列应用公式”。它比逐行循环更简洁，专门用于单列处理。

### 节点组成
- **列映射**（**控制**）也是一对节点：**MapColumnBegin** 和 **MapColumnEnd**。拖入后自动成对出现。

### 工作方式
- Begin 节点接收一个表格，并指定要映射的列（`col`）。它会将表格拆分为两部分输出：
  - **remains**：除指定列外的其他列组成的单行表格（实际上是一行，但包含多列）。
  - **cell**：当前行的指定列的值（类型取决于该列）。
- 在 Begin 和 End 之间，你可以对 `cell` 进行任意变换（例如数值运算、字符串处理），变换后的结果连接到 End。
- End 节点接收 **remains**（保持不变）和变换后的 **cell**，将它们重新组合成一行，并最终合并所有行输出。

## 4.4 单元格操作与行列转换（GetCell/SetCell/Unpack/Pack）

在循环或映射中，常常需要读取或修改表格中特定位置的值，或者将单行表格拆解为独立的值再重新组合。本节介绍四个节点：**提取单元格**、**更新单元格**、**解包**、**打包**。

### 提取单元格（GetCellNode）
- **分类**：**控制**
- **用途**：从单行表格中获取指定列的值。
- **参数**：**col** 选择列名；**row** 可选，指定行索引（默认 0，因为通常用于单行表格）。
- **输入**：`table`（单行表格），可选的 `row`（int）。
- **输出**：`value`（类型取决于列）。

### 更新单元格（SetCellNode）
- **分类**：**控制**
- **用途**：在单行表格中设置指定列的值，返回修改后的表格。
- **参数**：**col** 选择列名；**row** 可选。
- **输入**：`table`（单行表格），`value`（要设置的值，类型需匹配），可选的 `row`。
- **输出**：修改后的单行表格。

### 解包（UnpackNode）
- **分类**：**控制**
- **用途**：将单行表格拆解成多个独立的输出端口，每个端口对应一列的值。
- **参数**：**cols** 选择要解包的列名列表。
- **输入**：`row`（单行表格）。
- **输出**：
  - `unpacked_row`：与输入相同的单行表格，仅用于串联下游节点（如传递给打包节点）。
  - 为每个指定的列生成一个独立的输出端口，端口名即为列名，类型为该列的数据类型。
- **示例**：如果单行表格有“用户ID”和“评论内容”两列，解包后会输出两个独立的值，方便分别处理。

### 打包（PackNode）
- **分类**：**控制**
- **用途**：将多个输入值组合成一个单行表格。
- **参数**：**cols** 指定打包后的列名列表，与输入端口一一对应。
- **输入**：
  - `base_row`（可选）：基础单行表格，如果提供，则在其基础上添加新列；否则创建全新的单行表格。
  - 为每个列名定义一个输入端口，端口名即为列名，类型根据该列确定。
- **输出**：`packed_row`（单行表格），包含所有指定的列。
- **示例**：将处理后的“情感得分”和原始“用户ID”打包成一行表格，供 End 节点收集。

## 4.5 自定义脚本（CustomScriptNode）

当内置节点无法满足特定需求时，**自定义脚本** 节点允许你编写 Python 代码来处理数据。它提供了最大的灵活性。

### 节点特点
- **分类**：**控制**
- **动态端口**：你可以自定义输入端口和输出端口的名称和类型。
- **脚本编辑**：在节点配置中编写 Python 函数 `script`，该函数接收输入参数，返回输出值。
- **安全限制**：脚本运行在沙箱环境中，只能使用预定义的安全库（如 `math`、`re`、`json` 等），无法访问文件系统或网络。

### 配置步骤
1. 拖入 **自定义脚本** 节点。
2. 在节点参数中，定义输入端口（名称 + 类型）。
3. 定义输出端口（名称 + 类型）。
4. 在脚本编辑器中编写代码，格式如下：
   ```python
   def script(**inputs):
       # inputs 是一个字典，键为输入端口名
       # 处理逻辑
       result = some_function(inputs['port1'], inputs['port2'])
       return {'output_port': result}
   ```
5. 关闭脚本编辑器之后即可使用这个节点。

## 4.6 综合实战：用滚动窗口计算移动平均

下面通过一个简单但完整的例子，将本章的 **滑动窗口循环**、**统计信息**、**提取单元格**、**打包** 节点应用到实际数据中，计算股票收盘价的 5 日移动平均。

### 步骤

1. **获取数据**  
   - 拖入 **K线数据** 节点（**输入**），配置股票代码（如 `"TSLA"`）、时间间隔 `"1d"`、起止日期（例如 `2024-01-01` 到 `2024-03-31`）。  
   - 节点输出包含 `Open time` 和 `Close` 等列的表格。

2. **筛选列**（可选）  
   - 拖入 **筛选列** 节点（**表格处理**），只保留 `Open time` 和 `Close` 两列，简化数据。

3. **滑动窗口计算移动平均**  
   - 拖入 **滑动窗口循环**（**控制**），设置窗口大小 `5`，将筛选后的表格连接到 Begin 的 `table`。  
   - 在循环体内：  
     - 拖入 **统计信息** 节点（**分析**），将窗口表格（`window`）连接到它的 `table`，选择 `Close` 列，获取该窗口的均值（`mean` 输出端口）。  
     - 拖入 **提取单元格** 节点（**控制**），从窗口表格中提取最后一行的 `Open time` 作为该窗口的日期（因为窗口是滑动的，我们通常将结果对齐到窗口末尾）。  
     - 拖入 **打包** 节点（**控制**），将提取的日期（作为 `date`）和统计信息输出的 `mean`（作为 `ma5`）打包成一行表格。注意打包时 `cols` 参数应设置为 `["date", "ma5"]`，并将两个值分别连接到对应输入端口。  
   - 将打包后的单行表格连接到 End 节点的 `window` 输入。

4. **查看结果**  
   - End 节点输出一个表格，包含两列：`date`（窗口结束日期）和 `ma5`（5日移动平均值）。  
   - 双击 End 节点，即可预览计算结果。

5. **可视化对比**（可选）  
   为了直观对比原始收盘价与 5 日移动平均线，可以将两者合并到一张图表中。步骤如下：

   - **合并表格**  
     由于原始收盘价表格（步骤 2 输出）和移动平均表格（步骤 4 输出）都包含日期列（`Open time` 和 `date`），我们可以用 **连接** 节点将它们按日期对齐。  
     1. 从 **表格处理** 分类拖入 **连接** 节点。  
     2. 将步骤 2 的 `selected_table`（原始收盘价）连接到 **连接** 节点的 `left_table` 输入端口。  
     3. 将步骤 4 End 节点输出的移动平均表格连接到 **连接** 节点的 `right_table` 输入端口。  
     4. 配置 **连接** 节点参数：  
        - **left_on**：选择左表的 `Open time`（原始日期列）。  
        - **right_on**：选择右表的 `date`（移动平均表格的日期列）。  
        - **how**：选择 `INNER`（内连接，只保留两边都有的日期，也可根据需求选 `LEFT` 等）。  
     5. 节点输出 `joined_table`，包含 `Open time`、`Close`、`ma5` 三列。

   - **绘制双线图**  
     1. 从 **可视化** 分类拖入 **快速绘图** 节点。  
     2. 将 `joined_table` 连接到 **快速绘图** 的 `input` 端口。  
     3. 配置节点参数：  
        - **x_col**：选择 `Open time`（作为 X 轴）。  
        - **y_col**：创建两个Y轴，分别选择 `Close` 和 `ma5`。  
        - **plot_type**：为两个 Y 轴系列都选择 `line`（折线图）。  
        - 其他参数（如标题）可留空或自定义。  
     4. 运行后双击节点，即可看到收盘价与 5 日均线的对比曲线。

   如果希望两条线使用不同 Y 轴（例如收盘价用左轴，移动平均用右轴），可以在 **快速绘图** 节点中将 `y_axis` 分别设为 `left` 和 `right`。

>  这个例子展示了 **滑动窗口循环** 的核心用法——对每个窗口独立计算，并将结果重新组合。你可以替换窗口大小或统计方法（如 `std`、`max`），轻松实现滚动标准差、滚动最大值等功能。

##  小结

本章介绍了让流程具备逻辑控制能力的核心节点：
- **表格逐行循环**：对每一行独立处理，适合行级操作。
- **滑动窗口循环**：对连续窗口进行批量计算，适合时间序列。
- **列映射**：高效地对单列进行逐单元格变换。
- **提取/更新单元格** + **解包/打包**：在循环体内精细操作数据，实现行列转换。
- **自定义脚本**：用 Python 实现任意逻辑，突破内置节点限制。

掌握这些节点后，你的 NodePy 流程将不再局限于简单的线性变换，而是能够处理复杂的业务逻辑、实现自动化数据管道。下一章我们将进入 **机器学习实战**，带你用节点搭建完整的预测流程。
